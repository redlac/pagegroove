PAGEGROOVE

Privacy Policy

Effective date: April 2, 2026

PageGroove is a Chrome extension that turns the structure of the current web page into a deterministic synth loop. This policy explains what data the extension handles and how it is used.

What PageGroove handles

To generate audio for the current page, PageGroove reads structural information from the page you are viewing. This can include page metrics such as heading count, link count, image count, DOM depth, node count, hostname, page title, and a stable hash derived from page structure.

PageGroove also stores user settings such as playback state, volume, tempo, density, mood, and root note using Chrome extension storage.

How PageGroove uses data

  • To generate a deterministic audio loop for the current page
  • To keep playback synced with the active tab
  • To remember the user’s playback and sound settings

What PageGroove does not do

  • It does not send page content, browsing activity, or settings to the developer’s servers
  • It does not sell user data
  • It does not use data for advertising, profiling, or analytics
  • It does not share user data with third parties

Where data is processed

PageGroove processes page structure locally in the browser. Audio generation is performed locally with the Web Audio API.

Data retention

Settings saved by the extension remain in Chrome extension storage until the user changes them or removes the extension. Page structure is used to generate audio and is not stored remotely by the developer.

Permissions

PageGroove requests access to websites so it can read structural page information for the user-facing audio feature. It also uses tabs, storage, offscreen, and scripting to support playback, saved settings, and dynamic page handling.

Changes

This policy may be updated if the extension’s behavior changes. Any material changes should be reflected in an updated effective date and published version.
